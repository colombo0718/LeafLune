## 一、起點：兩個被濫用的詞、用核心需求重新錨

### 批判性思考

| | 內容 |
|---|---|
| 通俗誤解 | 看誰都不順眼、見人就鬥、批鬥成癮 |
| **LL 定義** | 能看到底層核心需求、以此為標準、剝除多餘不需要的東西 |
| 關鍵差別 | **有錨點** vs 純情緒反應 |

### 獨立思考

| | 內容 |
|---|---|
| 通俗誤解 | 自私、閉門造車、不聽人話 |
| **LL 定義** | 有清晰思考方向、思維不受外界影響人云亦云 |
| 關鍵差別 | **有方向** vs 純孤僻 |

### 「沒有對錯、只有立場」

| | 內容 |
|---|---|
| 通俗誤解 | 相對主義廢話 |
| **LL 定義** | 每人有各自視角與重視的痛點、推導出的結論不同、不是誰錯 |
| 關鍵差別 | **承認立場合理性** vs 不可知論 |

→ 三者共通：都需要「**核心需求錨點**」。沒錨、所謂思考都變情緒姿態。

## 二、數學類比：背公式 vs 推導

```
背公式型學生（學渣）         vs    推導型學生（學霸）
└─ 認知負擔 = O(公式數)              └─ 認知負擔 = O(公理數) << O(公式數)
└─ 新題型 → 卡住、要再背             └─ 新題型 → 從公理現場推、不卡

背 LL 結構型 colombo     vs    LL 因果推導型 colombo
└─ 記 26 個 repo 各幹嘛              └─ 記 LL 核心痛點 + 推導法則
└─ 半年後 50 個 → 腦袋爆             └─ 50 → 500 → 5000、O(1)、不爆
```

LL 規模化的關鍵差別：
- **直接背**：認知負擔 O(N²)（N 個 repo + N² 關係）、不可規模化
- **從痛點推導**：認知負擔 O(K)（K 條核心痛點 + 推導法則）、可規模化

## 三、LL 使命作為終極錨點

```
工具層：批判性思考 / 第一性原理 / 因果推導
    服務於 ↓
方法層：強化式學習框架 + 階梯式學習路徑 + 心流境界
    服務於 ↓
LL 使命：「學習變有趣、玩樂有價值」
    服務於 ↓
社會痛點：學習與玩樂的人為割裂
                填鴨背誦不快樂 + 玩樂被視為負面無價值
    映射到 ↓
個人需求：colombo 賺錢吃飯養家 + 創業生命續存
    根本層 ↓
企業作為生命個體：向外提供價值（學習變有趣）
                  向內續存自己（colombo 持續推進）
```

→ **工具不是目的、推進 LL 才是**。任何決策都該回溯這條鏈、找定位。

## 四、思考的工法（三步、不繁瑣）

第一性原理三步、對應 CAPTAIN.md §2.1 設計閘門：

```
🚦 1. 核心需求是什麼？（拎出一句話、不含預設方案）
🚦 2. 候選解有哪些？（列 3-5 個、不要在 step 1 鎖死）
🚦 3. 附加條件當篩網？（即時？autonomous？透明？...）
   → 篩到剩 1-2 個 = 最優解
```

每次 AI 要設計 / 規劃 / 架構新東西時、colombo（或 AI 自己）跑這三步。
答不出 step 1、就退回「20 行 MVP」級。

## 五、兩個要避免的坑

### 過度思考 → 完美主義 → 停滯

```
症狀：跑三步跑太徹底、列 8 個候選、要 colombo 補更多 context、
      想清楚所有 edge case 才動手、結果 1 小時什麼都沒做。
矯正：三步是篩網不是攔截、列 3 個候選夠、最優解一出來就推進。
      「先求有再求好」永遠優先。
```

### 消極 → 虛無主義

```
症狀：colombo 任何提案、反覆問「真有痛點嗎？」「真的最優嗎？」
      推到所有事都沒意義、什麼都不做。
矯正：使命錨點足夠、不要拿錨點去懷疑使命本身。
      LL 使命「學習變有趣、玩樂有價值」是公理、不是定理。
```

→ 兩個坑都是「**把工具當身份**」。批判性思考是工具、不是 LL 員工的人格認同。

## 六、應用層

### 個人認知

看新聞、做決定、跟人辯論、決策投資 → 先問核心需求。

### 專案 / 系統設計

設計閘門三問（CAPTAIN.md §2.1）+ 三步工法。

### 跨人協作（立場觀）

兩人核心需求不同 → 推導結論不同 → 不是誰錯。
- colombo（LL Edutainment Studio）vs cwsoft 同事（client work）：邊界、各自治理
- LL 內部治理 vs cwsoft 治理：兩條獨立鏈、不混用

### AI 協作

AI 訓練資料偏「完整框架」、預設違反此原則。
- colombo 的角色 = 守錨點、AI 想堆 framework 時拉回核心需求
- AI 的協作守則 = 不當應聲蟲、不當鬥雞、贊同 / 反對都附 reasoning chain
